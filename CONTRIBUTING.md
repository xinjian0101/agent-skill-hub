# Contributing

Read the README, About, and Roadmap before contributing.

Use an Issue for large changes. Keep each pull request focused and include the problem, intended result, examples, validation steps, compatibility impact, attribution, and documentation updates.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review of submitted material. Do not publish private information or material that you do not have permission to distribute.
