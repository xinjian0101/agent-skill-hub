# Maintenance Policy

## Project phase

Agent Skill Hub is currently in the foundation phase. Repository documentation describes planned scope and must not be presented as completed product behavior.

## Change requirements

- Keep changes focused and reviewable.
- Update documentation when formats or compatibility change.
- Add examples and repeatable checks for new entries.
- Preserve attribution and license information.
- Record known limitations instead of hiding them.

## Review cadence

The roadmap should be reviewed whenever a milestone changes state. Examples, links, and compatibility notes should be checked before each release.

## Release standard

A public preview should include reproducible setup instructions, tested examples, accurate screenshots, clear limitations, and release notes that match the delivered repository state.
